⚙️ 数据源配置

{# ===== 说明 ===== #}
💡 使用说明：下方列出支持的所有数据采集平台。 RSS 源管理 → 其他平台需要配置对应凭证，配置完成后点击「保存」，即可在仪表盘启用采集。 敏感信息（Cookie、Token 等）仅本地存储，不会上传任何服务器。
{# ===== 配置表单 ===== #}
{# ===== 知乎 ===== #}
💬 知乎
📌 配置说明：知乎需要 Cookie 凭证。 登录知乎后，按 F12 打开开发者工具 → Application → Cookies → 复制 z_c0 的值填入。 支持抓取用户文章、回答、收藏夹内容。
{# ===== 小红书 ===== #}
🌸 小红书
📌 配置说明：小红书需要 Cookie + XHS Token。 登录小红书网页版后，在开发者工具 → Application → Cookies 中找到 web_session 和 xhs-token。 支持按用户或关键词采集笔记。
{# ===== YouTube ===== #}
▶️ YouTube
📌 配置说明：YouTube 需要 YouTube Data API v3 的 API Key。 在 Google Cloud Console 创建项目并申请 API Key，启用 YouTube Data API v3。 支持按频道或搜索关键词抓取视频标题、描述、字幕。

🔍 关键词搜索

{# ===== Twitter/X ===== #}
🐦 Twitter / X
📌 配置说明：Twitter 需要 Bearer Token。 登录 Twitter 后，在开发者工具 → Network 中找任意 API 请求，复制 Request Headers 中的 Authorization: Bearer ... 值。 支持按用户推文或关键词搜索采集。
{# ===== 抖音 ===== #}
🎵 抖音
📌 配置说明：抖音需要 Cookie 凭证。 登录抖音网页版后，通过开发者工具获取 Cookie。 支持按用户主页或关键词采集视频标题和描述。
{# ===== Sitemap ===== #}
🗺️ Sitemap
📌 配置说明：Sitemap 适合采集任何支持 sitemap.xml 的网站。 输入网站地图地址，系统自动解析并抓取所有链接的文章内容。 适合采集技术博客、新闻网站等内容型站点。
{# ===== 自定义 API ===== #}
🔌 自定义 API
📌 配置说明：自定义 API 支持接入任意内容接口。 输入接口地址，支持 GET/POST 方式，系统自动解析返回的文章列表。 返回格式支持 JSON 数组（每项含 title、content、url 字段）。
