## 概述

本版本从 Y2A-Auto 项目移植经生产验证的 LLM 响应处理工具和 YouTube 错误诊断机制，修复了两个反复出现的已知问题：

1. ❌ **LLM 改写反复输出英文推理内容** → ✅ **三层推理防御架构**
2. ❌ **YouTube 报错信息不明确** → ✅ **分级错误诊断**

## 技术架构

```
用户输入 ──→ clean_input_content() ──→ LLM 请求 + thinking:disabled
                                           │
                      ┌────────────────────┘
                      ↓
                   LLM 响应
                      │
              strip_reasoning_thoughts()
                      │
              _is_reasoning_content() ──→ yes → _extract_final_from_reasoning()
                      │ no                          │
                      ↓                             ↓
              提取最终正文 ─────────────→ rewrite result
```

### 三层推理防御

| 层 | 触发时机 | 策略 |
|----|----------|------|
| 第1层: 请求层 | 请求发送前 | 自动添加 `extra_body.thinking = disabled` |
| 第2层: 响应层 | 收到 LLM 响应 | `strip_reasoning_thoughts()` + `_is_reasoning_content()` |
| 第3层: 降级层 | API 报错 | 检测 unknown/unsupported → 移除参数重试 |
