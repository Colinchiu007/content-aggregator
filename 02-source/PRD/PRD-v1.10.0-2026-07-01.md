## 一、背景与目标

### 1.1 背景

当前 content-aggregator 的改写服务 (rewrite.py) 支持 4 种改写风格（轻松易懂/正式严谨/吸引眼球/深度分析），缺乏**认知引导型**的改写能力。同时，AGENTS.md 需要规范化知识分层。

### 1.2 目标

- 新增「认知锚点」改写风格，提供结构化认知引导的改写能力
- AGENTS.md 知识分层：主文件只保留流程框架，详细规范拆分到 references/

---

## 二、功能详述

### 2.1 新增改写风格：认知锚点

**名称**: `"认知锚点"`

**定位**: 第 5 种改写风格，与原有的 4 种（轻松易懂/正式严谨/吸引眼球/深度分析）并列。

**5 种认知锚点类型**:

| 锚点类型 | 格式标记 | 说明 |
|---------|---------|------|
| 核心判断 | 【核心判断】 | 文章的核心观点或关键结论，一句话提炼 |
| 断点 | 【断点】 | 认知转折点，读者原有认知被打破或刷新之处 |
| 分流 | 【分流】 | 不同观点、路径或可能性的分岔点 |
| 对比 | 【对比】 | 两个事物、时期或观点的对比，突出差异 |
| 变化 | 【变化】 | 趋势、状态或认知的变化轨迹 |

**输出规范**:
- 以段落正文为主体，每嵌入一个锚点用【】标记
- 保持流畅可读，不写成清单或表格
- 锚点分布均匀，每 500 字左右至少出现 2-3 个锚点
- 保持原文核心信息和数据准确

**集成方式**: 在 `backend/app/services/rewrite.py` 的 `STYLE_PROMPTS` 字典中注册。`rewrite_content()` 函数自然支持新的风格选择。

### 2.2 AGENTS.md 知识分层

## 六、LLM 选型与约束

### 6.1 模型选型矩阵

| 模型 | 用途 | 优先级 | 调用方式 | 备注 |
|------|------|--------|----------|------|
| **DeepSeek-V3** | 主力改写模型 | P0 | API (HTTP) | 性价比最优，中文改写质量高 |
| **通义千问-Plus** | 备用改写模型 | P1 | API (HTTP) | DeepSeek 不可用时降级使用 |
| **DeepSeek-R1** | 深度分析改写 | P1 | API (HTTP) | 需要推理能力的改写场景 |
| **GPT-4o-mini** | 轻量改写/摘要 | P2 | API (HTTP) | 简单改写任务，成本最低 |
| **本地 Ollama (Qwen2.5-7B)** | 离线兜底 | P3 | Local | 所有 API 不可用时的降级方案 |

### 6.2 调用约束

| 约束项 | 值 | 说明 |
|--------|-----|------|
| **单次调用超时** | 60s | 超时后自动重试或降级 |
| **最大重试次数** | 3 | 指数退避（1s -> 2s -> 4s） |
| **并发调用上限** | 5 | 防止 API 限流 |
| **单次最大 token** | 8192 | 输出 token 上限 |
| **日调用预算** | 50,000 tokens/用户 | 超限后降级到本地模型 |
| **月预算上限** | 500 CNY/月（全量用户） | 超限后暂停非核心改写功能 |

### 6.3 质量保证机制

1. **输出校验**：改写结果必须通过 `len(output) >= len(input) * 0.5` 最小长度检查
2. **格式校验**：认知锚点风格需检查标记出现频率（每 500 字 >= 2 个）
3. **敏感词过滤**：改写结果经 DFA 敏感词库过滤
4. **降级策略**：模型返回异常或质量不达标时，自动降级到下一优先级模型
5. **人工抽检**：每日随机抽检 10% 改写结果，用于质量评估

### 6.4 API 认证机制

- 所有 API Key 通过环境变量注入：`DEEPSEEK_API_KEY`、`QWEN_API_KEY`、`OPENAI_API_KEY`
- 禁止在代码中硬编码任何 API Key
- 本地 `.env` 文件不进入版本控制（已在 `.gitignore` 中配置）
