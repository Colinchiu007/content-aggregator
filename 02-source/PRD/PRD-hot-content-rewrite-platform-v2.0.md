## 1. 产品概述

### 1.1 产品定位

**一句话描述**：
> 帮助内容创作者快速将优质文章/文案进行AI改写，并一键发布到多个平台的内容生产力工具。

**核心价值主张**：
- 🎨 **降低创作成本**：不再从零开始写，基于优质内容改写
- ⏱️ **提升发布效率**：采集→改写→发布，一站式完成
- 🎭 **多风格适配**：同一内容，不同平台用不同风格

### 1.2 目标用户

| 用户类型 | 占比 | 核心痛点 | 使用频率 |
|---------|------|---------|---------|
| **自媒体运营** | 40% | 需要每天发布多篇文章，创作压力大 | 每日 |
| **企业内容团队** | 30% | 需要批量生产内容，保持品牌调性 | 每周 |
| **个人博主** | 20% | 想蹭热点，但不知道怎么"洗稿" | 每周 |
| **营销人员** | 10% | 需要快速生成营销文案 | 按需 |

### 1.3 产品边界

**包含**：
- ✅ 内容采集（URL、粘贴、文件）
- ✅ AI改写（多风格、多长度）
- ✅ 多平台发布（微信公众号、知乎等）
- ✅ 基础管理（历史记录、账号配置）

**不包含**（Phase 1）：
- ❌ 原创内容创作（只做改写，不做从零创作）
- ❌ 全量内容聚合（不做内容平台，只做工具）
- ❌ 图片/视频处理（Phase 2再做）
- ❌ 团队协作（Phase 3再做）

### 1.4 成功指标

**核心KPI**：
| 指标 | MVP目标 | Phase 2目标 | Phase 3目标 |
|------|---------|-------------|-------------|
| 注册用户 | 500+ | 2000+ | 5000+ |
| 日活用户 | 50+ | 200+ | 500+ |
| 每周改写次数 | 200+ | 1000+ | 3000+ |
| 发布成功率 | >95% | >98% | >99% |
| 用户满意度 | 4.0+ | 4.3+ | 4.5+ |

## 2. 用户角色和使用场景

### 2.1 用户角色画像

#### 角色1：自媒体运营小王

**背景**：
- 28岁，运营科技类公众号+知乎专栏
- 每天需要发布1-2篇文章
- 痛点：创作压力大，经常熬夜写稿

**使用场景**：
```
场景：竞品分析+快速跟进
1. 早上浏览时发现竞品发布了一篇10W+爆款
2. 复制文章链接 → 打开我们的平台
3. 粘贴链接 → 系统自动采集全文
4. 选择"深度分析"风格 → 点击"开始改写"
5. 5分钟后，获得改写后的文章
6. 简单调整后 → 一键发布到自己的公众号和知乎
7. 当天下午发布，阅读量达到5000+
```

**核心需求**：
- 快速采集竞品文章
- 高质量改写（不被判定为抄袭）
- 多平台一键发布

---

#### 角色2：企业内容团队负责人小李

**背景**：
- 某SaaS公司内容营销负责人
- 需要每周生产3-5篇高质量文章
- 痛点：团队人力有限，内容产出慢

**使用场景**：
```
场景：批量内容生产
1. 周一团队会议，确定本周3个选题
2. 让团队成员采集行业优质文章（3-5篇/选题）
3. 全部导入系统，批量改写
4. 统一调整为"正式严谨"风格（符合品牌调性）
5. 安排发布计划：周三、周五各发布一篇
6. 系统定时自动发布到公司公众号和官网博客
```

**核心需求**：
- 批量处理能力
- 品牌语调一致性（改写模板）
- 定时发布

---

#### 角色3：个人博主小张

**背景**：
- 业余时间运营个人技术博客
- 想蹭热点，但不知道怎么"洗稿"
- 痛点：担心版权问题，不知道如何合理引用

**使用场景**：
```
场景：热点跟进
1. 在知乎发现一个热门话题（AI相关）
2. 点击"收藏到素材库"（只存标题+链接）
3. 周末有时间时，打开素材库
4. 选择那篇热文 → 点击"一键改写"
5. 选择"轻松易懂"风格（适合个人博客）
6. 简单编辑后发布到个人博客
7. 在文章末尾注明"灵感来源：XXX"
```

**核心需求**：
- 素材库功能（收藏优质内容）
- 合理的改写（保留来源标注）
- 简单易用（不需要复杂配置）

### 3.2 功能详细需求

#### 功能模块1：内容输入

##### 3.2.1 URL采集

**功能描述**：用户输入文章URL，系统自动采集全文内容。

**输入**：
- URL输入框（支持各大内容平台）
- 示例：
  - 微信公众号：`https://mp.weixin.qq.com/s/xxx`
  - 知乎专栏：`https://zhuanlan.zhihu.com/p/xxx`
  - 掘金：`https://juejin.cn/post/xxx`
  - 其他：任意支持HTTP访问的URL

**处理流程**：
1. 验证URL格式
2. 检测平台类型（自动识别或手动选择）
3. 调用对应平台的采集器
4. 提取正文、标题、作者、发布时间等元数据
5. 转换为Markdown格式
6. 缓存结果（24小时内重复请求直接返回缓存）

**输出**：
- 采集成功：
  - 显示文章标题、作者、字数、来源
  - 显示内容预览（前500字符）
  - 提供"重新采集"和"手动编辑"按钮
- 采集失败：
  - 显示错误原因（404、反爬、超时等）
  - 提供"重试"按钮
  - 建议用户"粘贴文本"作为备选方案

**验收标准**：
- [ ] 支持微信公众号、知乎、掘金等主流平台
- [ ] 采集成功率 >90%（在网络正常的情况下）
- [ ] 采集速度 <10秒（普通网页）、<30秒（需要渲染的页面）
- [ ] 内容提取准确率 >85%（正文完整，无广告、无多余内容）

**异常情况处理**：
| 异常情况 | 处理方式 |
|---------|---------|
| URL格式错误 | 提示"请输入正确的URL" |
| 404/页面不存在 | 提示"文章已删除或链接错误" |
| 反爬拦截 | 提示"该网站有反爬限制，建议粘贴文本" |
| 超时（>30秒） | 提示"采集超时，请稍后重试" |
| 内容提取失败 | 提示"无法识别文章内容，请手动粘贴" |

---

##### 3.2.2 粘贴文本

**功能描述**：用户直接粘贴文章内容，系统自动识别格式并处理。

**输入**：
- 文本输入区域（支持大段文本）
- 格式自动识别：
  - 纯文本（Plain Text）
  - HTML（网页源码）
  - Markdown

**处理流程**：
1. 自动检测文本格式（或允许用户手动选择）
2. 如果是HTML，转换为Markdown
3. 如果是纯文本，智能分段（双换行为段落分隔）
4. 清理无效字符、标准化换行符
5. 尝试提取标题（第一行或第一个#标题）

**输出**：
- 标准化后的Markdown内容
- 显示提取的标题、字数统计
- 提供"手动调整"入口

**验收标准**：
- [ ] 支持纯文本、HTML、Markdown三种格式
- [ ] 格式识别准确率 >95%
- [ ] HTML转Markdown保留基本格式（标题、段落、列表、链接、图片）

---

##### 3.2.3 文件上传

**功能描述**：用户上传本地文件，系统解析文件内容。

**支持格式**：
- Word文档（.docx）
- PDF文档（.pdf）
- 纯文本（.txt）
- Markdown（.md）
- HTML网页（.html, .htm）

**输入**：
- 文件选择对话框
- 拖拽上传区域

**处理流程**：
1. 检查文件格式（扩展名+文件头）
2. 调用对应格式的解析器
3. 提取正文内容
4. 转换为Markdown格式
5. 提取元数据（标题、作者、创建时间）

**输出**：
- 解析成功：显示文件内容预览
- 解析失败：提示不支持的格式或文件损坏

**验收标准**：
- [ ] 支持上述5种格式
- [ ] Word解析准确率 >90%（保留段落、标题、表格）
- [ ] PDF解析准确率 >85%（纯文本PDF，扫描版暂不支持）
- [ ] 文件大小限制：<20MB

**异常情况处理**：
| 异常情况 | 处理方式 |
|---------|---------|
| 格式不支持 | 提示"暂不支持该格式，请转换为TXT或粘贴文本" |
| 文件过大（>20MB） | 提示"文件过大，请上传<20MB的文件" |
| 解析失败 | 提示"文件解析失败，可能已损坏，请重新上传" |
| 密码保护的PDF/Word | 提示"不支持加密文件，请移除密码后重试" |

#### 功能模块2：AI改写

##### 3.2.4 改写风格选择

**功能描述**：用户选择改写后的文章风格。

**可选风格**：
| 风格 | 描述 | 适用场景 |
|------|------|---------|
| 轻松易懂 | 语言活泼，通俗易懂 | 社交媒体、个人博客 |
| 正式严谨 | 用语规范，逻辑清晰 | 官网、报告、企业公众号 |
| 吸引眼球 | 标题党，抓人眼球 | 头条号、自媒体 |
| 深度分析 | 加入个人见解，深度剖析 | 知乎、公众号长文 |
| 自定义 | 用户自己输入改写要求 | 高级用户 |

**交互设计**：
- 单选按钮（只能选一个）
- 每个风格有详细说明和示例
- 选择后实时显示"预计改写时间"（如：约5分钟）

**验收标准**：
- [ ] 至少提供4种预设风格
- [ ] 每种风格有明确的示例（点击可查看）
- [ ] 自定义Prompt支持（高级选项）

---

##### 3.2.5 改写长度选择

**功能描述**：用户选择改写后的文章长度。

**可选项**：
- 保持原长（±10%）
- 压缩精简（-30%，适合摘要）
- 扩展详细（+30%，适合深度文章）

**交互设计**：
- 单选按钮
- 显示预计字数（如：原文1234字 → 改写后约800字）

**验收标准**：
- [ ] 三种长度选项都可用
- [ ] 实际字数与预计字数的偏差 <20%

---

##### 3.2.6 高级选项

**功能描述**：用户提供额外的改写要求。

**可选项**：
- ☑ SEO优化（自动添加关键词、优化标题）
- ☑ 保留关键数据（不改变数字、日期、专有名词）
- ☑ 添加来源标注（在文末注明"灵感来源：XXX"）
- ☑ 生成摘要（在文章开头添加100字摘要）

**自定义Prompt**：
- 文本输入框（可选）
- 示例："请用更口语化的表达，加入一些幽默元素"

**保存为模板**：
- 按钮："保存为模板"
- 输入模板名称
- 下次可直接使用模板

**验收标准**：
- [ ] 所有选项都能正确传递给AI
- [ ] 自定义Prompt字数限制：<500字
- [ ] 模板保存和加载功能正常

---

##### 3.2.7 改写处理

**功能描述**：系统调用AI模型进行改写。

**处理流程**：
1. 接收用户输入的内容和改写选项
2. 构建Prompt（包含风格、长度、高级选项）
3. 调用LLM Client（接入现有的AI模型）
4. 显示进度条（模拟进度或根据token生成速度估算）
5. 改写完成后，返回改写后的内容

**进度提示**：
```
正在改写...
[==========          ] 50%
预计剩余时间：3分钟
```

**改写结果**：
- 显示原文vs改写后的对比（左右分栏）
- 原文可折叠（默认折叠）
- 改写后内容可编辑（双击或点击"编辑"按钮）

**验收标准**：
- [ ] 改写速度：1000字文章 <5分钟（取决于AI模型）
- [ ] 改写质量：人工评估，满意度 >80%
- [ ] 支持中断和重新改写

---

##### 3.2.8 改写历史

**功能描述**：用户查看过去的改写记录。

**显示内容**：
- 改写时间
- 原文标题
- 改写风格
- 发布状态（已发布/未发布）

**操作**：
- 查看详情（回到改写结果页面）
- 重新发布
- 删除记录

**验收标准**：
- [ ] 保存最近100条改写记录
- [ ] 支持按时间、风格、发布状态筛选

#### 功能模块3：多平台发布

##### 3.2.9 账号绑定

**功能描述**：用户绑定要发布到的平台账号。

**支持平台**（Phase 1）：
- 微信公众号（需要AppID + AppSecret）
- 知乎（需要用户名 + 密码 或 OAuth）

**绑定流程**（以微信公众号为例）：
1. 用户点击"绑定微信公众号"
2. 输入AppID和AppSecret（或扫码授权）
3. 系统验证凭据
4. 绑定成功，显示公众号名称+头像

**管理**：
- 查看已绑定的账号
- 解绑
- 重新授权

**验收标准**：
- [ ] 支持至少2个主流平台
- [ ] 凭据加密存储（不明文保存密码）
- [ ] 绑定成功率 >95%

---

##### 3.2.10 平台适配

**功能描述**：根据不同平台的格式要求，自动调整内容格式。

**平台格式要求**：
| 平台 | 格式 | 特殊要求 |
|------|------|---------|
| 微信公众号 | HTML | 需要排版（段落间距、字体） |
| 知乎 | Markdown | 支持代码块、公式 |
| 头条号 | HTML | 图片需要上传到平台 |

**适配处理**：
- 微信公众号：Markdown转HTML + 添加CSS样式
- 知乎：保持Markdown格式
- 头条号：下载图片 → 上传到头条 → 替换图片链接

**验收标准**：
- [ ] 各平台格式正确，无乱码
- [ ] 图片处理成功（如果需要）
- [ ] 发布后显示效果与预览一致

---

##### 3.2.11 一键发布

**功能描述**：用户选择目标平台，系统自动发布。

**发布流程**：
1. 用户选择要发布的平台（可多选）
2. 选择发布时间（立即发布 或 定时发布）
3. 点击"一键发布"
4. 系统依次发布到各平台
5. 显示发布进度和结果

**发布结果**：
```
正在发布...
✅ 微信公众号：发布成功
  文章链接：https://mp.weixin.qq.com/s/xxx
✅ 知乎：发布成功
  文章链接：https://zhuanlan.zhihu.com/p/xxx
⏳ 头条号：发布中...
```

**验收标准**：
- [ ] 支持多平台同时发布
- [ ] 发布成功率 >95%（网络正常情况下）
- [ ] 发布失败后自动重试（最多3次）

---

##### 3.2.12 定时发布

**功能描述**：用户设置未来某个时间自动发布。

**设置界面**：
- 发布时间选择器（日期+时间）
- 时区选择（默认Asia/Shanghai）

**处理流程**：
1. 用户设置定时发布
2. 系统将任务加入发布队列
3. 到达指定时间后，自动执行发布
4. 发布成功后，发送通知（站内信/邮件）

**验收标准**：
- [ ] 定时发布时间误差 <5分钟
- [ ] 支持取消定时发布
- [ ] 系统重启后，定时任务不丢失

---

##### 3.2.13 发布日志

**功能描述**：用户查看历史发布记录。

**显示内容**：
- 发布时间
- 发布平台
- 文章
