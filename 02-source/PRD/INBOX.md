# PRD 需求收件箱

> ⚠️ **任何 Agent 或用户都可以追加需求到这个文件**，不限 Session、不限身份。
> CEO 会每 4 小时检查并合并到正式 PRD。

---

## 使用说明

### 谁可以写？

✅ **任何 Agent**（PM、CTO、Specialist、CEO）
✅ **用户本人**（在任何对话里）
❌ **不需要知道 CEO 在哪个 Session 在线**

### 怎么写？

在文件末尾的 `## 待处理需求` 章节下**追加**新需求（不要覆盖已有内容）：

## 使用说明

### 追加新需求

在文件末尾的 `## 待处理需求` 章节下追加：

<pre>
## [YYYY-MM-DD HH:MM] 需求标题
- **来源**：（PM 会话 / CTO 会话 / 用户直接提出）
- **描述**：...
- **优先级**：P0/P1/P2
- **提出人**：用户名
- **相关文件**：（可选，如 `web/server.py`）
</pre>

### 注意事项

1. **追加，不要覆盖**：在 `## 待处理需求` 章节末尾追加，不要删除别人写的需求
2. **格式必须一致**：按上面的格式写，方便 CEO 自动解析
3. **不要手动修改正式 PRD**：正式 PRD 只有 CEO 能改，其余人通过 INBOX 提需求
4. **大需求可以附加文件**：如果需求描述很长，可以写到 `team/projects/PROJECT-XXX/PRD/INBOX-附件.md`，然后在 INBOX 里贴链接
