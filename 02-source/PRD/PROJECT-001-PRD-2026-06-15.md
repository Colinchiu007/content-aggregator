## 1. 产品概述

### 1.1 产品定位

**一句话描述**：
> 帮助内容创作者快速将优质文章/文案进行AI改写，并一键发布到多个平台的内容生产力工具。

**核心价值主张**：
- 📝 **降低创作成本**：不再从零开始写，基于优质内容改写
- ⚡ **提升发布效率**：采集→改写→发布，一站式完成
- 🎨 **多风格适配**：同一内容，不同平台用不同风格
- 🔥 **抢占热点**：热榜发现+快速改写，30分钟内发布

**产品边界**：
- ✅ 做：内容采集（按需）、AI改写、多平台发布
- ❌ 不做：全量内容聚合、原创内容写作、图片/视频生成

### 1.2 产品目标

**商业目标**：
- 6个月内获取5000注册用户，日活500+
- 12个月内付费转化率8%，ARPU ¥80+
- 成为"内容改写"领域的头部工具

**用户目标**：
- 节省70%的内容创作时间
- 提升内容质量（AI优化）
- 提高多平台分发效率

**产品目标**：
- 改写质量：AI改写后的人工编辑率 <30%
- 发布成功率：>95%
- 用户满意度：4.3/5.0

### 1.3 产品路线图

| 阶段 | 时间 | 核心功能 | 目标 |
|------|------|---------|------|
| **MVP** | 第1-3周 | URL采集+AI改写+发布 | 可用，能跑通流程 |
| **内测** | 第4周 | 邀请10个用户测试 | 收集反馈，修复bug |
| **公测** | 第5-8周 | 开放注册，基础运营 | 获取前500用户 |
| **Phase 2** | 第9-12周 | 热榜 ✅ + 素材库 ⏳ + 竞品监控 ✅ | 提升留存率 |
| **Phase 3** | 第13-18周 | 团队协作+智能推荐+商业化 | 探索盈利模式 |

## 2. 市场分析

### 2.1 市场规模

**目标市场规模**：
- 自媒体从业者：500万+（公众号、知乎、头条等）
- 企业内容团队：50万+（需要批量生产内容）
- 个人博主：2000万+（有内容创作需求）

**可触达市场（TAM）**：
- 付费意愿用户：100万+（自媒体+企业）
- 客单价：¥29-199/月
- 市场规模：¥3.5亿-20亿/年

### 2.2 竞品分析

#### 竞品对比矩阵

| 维度 | 135编辑器 | 秀米 | Notion AI | Feedly | **我们的产品** |
|------|-----------|------|-----------|---------|--------------|
| **内容采集** | ❌ | ❌ | ❌ | ✅ | ✅ (按需) |
| **AI改写** | ❌ | ❌ | ✅ | ❌ | ✅ (多风格) |
| **多平台发布** | ❌ | ❌ | ❌ | ❌ | ✅ |
| **热榜发现** | ❌ | ❌ | ❌ | ✅ | ✅ (Phase 2) |
| **素材库** | ❌ | ✅ (模板) | ❌ | ✅ | ✅ (Phase 2) |
| **定价** | ¥0-99/月 | ¥0-199/月 | ¥68/月 | $0-99/月 | ¥29-199/月 |

**竞争优势**：
1. **一站式**：采集+改写+发布，无需切换工具
2. **按需采集**：不建内容库，成本低、版权友好
3. **多风格改写**：适配不同平台，而非通用改写
4. **热榜联动**：发现热点→快速改写→抢占流量

**竞争劣势**：
1. **品牌知名度**：新品牌，需要时间积累
2. **改写质量**：需要持续优化AI模型
3. **平台覆盖**：初期支持的平台数量有限

### 2.3 用户痛点

**痛点1：创作效率低**
- 从零开始写一篇2000字文章需要3-5小时
- 多平台发布需要手动调整格式，额外1-2小时

**痛点2：改写质量差**
- 手动"洗稿"容易被平台判定为抄袭
- 改写后失去原文精髓，质量下降

**痛点3：热点跟进慢**
- 发现热点后，手动写作跟不上热点时效
- 竞品发布爆款后，无法快速跟进

**痛点4：工具分散**
- 需要同时使用：采集工具+写作工具+排版工具+发布工具
- 多工具切换，效率低、学习成本高

## 3. 用户画像与使用场景

### 3.1 用户画像

#### 用户类型1：自媒体运营者（个人）

**用户标签**：
- 年龄：25-35岁
- 职业：自媒体运营、自由撰稿人
- 平台：微信公众号、知乎、头条
- 痛点：每天需要发布1-3篇文章，创作压力大

**使用场景**：
```
场景：竞品分析+快速跟进
目标：发现竞品爆款文章后，快速产出类似内容

流程：
1. 在公众号发现竞品10W+文章
2. 复制文章链接 → 粘贴到系统
3. 选择改写风格（"轻松易懂"）
4. AI自动改写（保留核心观点，调整表达方式）
5. 一键发布到自己的公众号
6. 总耗时：15分钟（vs 手动3小时）
```

**核心需求**：
- 快速采集竞品文章
- 高质量改写（不被判定抄袭）
- 一键发布到多个平台

---

#### 用户类型2：企业内容团队

**用户标签**：
- 年龄：28-40岁
- 职业：企业内容营销、品牌公关
- 平台：公司官网、公众号、知乎机构号
- 痛点：需要批量生产内容，保持品牌调性

**使用场景**：
```
场景：多平台内容分发
目标：一篇内容，发布到多个平台（风格不同）

流程：
1. 采集公司官网的新闻稿（或粘贴文本）
2. 选择目标平台：
   - 微信公众号：正式风格
   - 知乎：深度分析风格
   - 头条：吸引眼球风格
3. AI自动生成3个版本
4. 分别发布到对应平台
```

**核心需求**：
- 品牌语调模板（统一风格）
- 团队协作（素材共享、权限管理）
- 批量发布（定时发布）

---

#### 用户类型3：个人博主

**用户标签**：
- 年龄：20-30岁
- 职业：学生、职场新人
- 平台：个人博客、知乎、B站专栏
- 痛点：想蹭热点，但不知道怎么"洗稿"

**使用场景**：
```
场景：热点跟进
目标：发现热榜文章后，快速改写成自己的内容

流程：
1. 浏览知乎热榜，发现"AI写作"话题热议
2. 点击热榜文章 → 系统自动采集全文
3. 选择"轻松易懂"风格（适合个人博客）
4. AI改写 → 手动微调（10分钟）
5. 发布到个人知乎账号
```

**核心需求**：
- 热榜发现（找到热点）
- 简化改写（降低创作门槛）
- 低成本（个人版定价敏感）

---

### 3.2 用户旅程地图

```
阶段1：发现
  用户发现竞品爆款 or 热榜话题
  ↓
阶段2：采集
  复制链接 or 粘贴文本 → 系统自动采集全文
  ↓
阶段3：改写
  选择改写风格 → AI自动改写 → 手动微调（可选）
  ↓
阶段4：发布
  选择目标平台 → 一键发布 or 定时发布
  ↓
阶段5：数据分析（Phase 3）
  查看阅读量、点赞量 → 优化下次改写策略
```

**关键痛点节点**：
- 阶段2：采集失败（反爬、格式混乱）
- 阶段3：改写质量差（需要大量手动调整）
- 阶段4：发布失败（平台API限制）

##### 功能1：URL采集

**功能描述**：
用户粘贴文章URL，系统自动采集全文内容。

**需求详情**：

| 需求ID | 需求描述 | 优先级 | 验收标准 |
|--------|---------|--------|---------|
| F-1-1 | 支持主流平台URL采集 | P0 | 支持公众号、知乎、掘金、头条 |
| F-1-2 | 显示采集进度 | P0 | 进度条+已采集字数 |
| F-1-3 | 采集结果预览 | P0 | 显示标题、作者、字数、发布时间 |
| F-1-4 | 重新采集按钮 | P1 | 采集失败后，一键重新采集 |
| F-1-5 | 手动编辑采集结果 | P1 | 采集不完整时，允许用户手动补充 |
| F-1-6 | 支持更多平台 | P2 | 支持CSDN、简书、Medium等 |

**用户界面**：
```
┌─────────────────────────────────────┐
│  输入文章链接                        │
│  ┌───────────────────────────────┐ │
│  │ https://zhuanlan.zhihu.com/p/ │ │
│  └───────────────────────────────┘ │
│  [采集全文]                         │
│                                     │

│  [粘贴文本] [上传文件]              │
```

**异常处理**：
- 采集失败：显示错误原因（404、反爬、网络超时等），提供"手动粘贴"备选方案

### 5.4 数据库设计

```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    created_at TIMESTAMP DEFAULT NOW(),
    subscription_type VARCHAR(20) DEFAULT 'free'  -- free/basic/pro/enterprise
);
```

**素材库表（materials）**：
```sql
CREATE TABLE materials (
    id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(id),
    title VARCHAR(200) NOT NULL,
    source_url TEXT,
    summary TEXT,
    cover_image TEXT,
    tags TEXT[],  -- 数组类型，存储标签
    created_at TIMESTAMP DEFAULT NOW()
);
```

**改写历史表（rewrite_history）**：
```sql
CREATE TABLE rewrite_history (
    id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(id),
    source_type VARCHAR(20),  -- url/text/file
    source_content TEXT,
    rewrite_style VARCHAR(50),
    rewrite_length VARCHAR(20),
    result_content TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);
```

**发布记录表（publish_logs）**：
```sql
CREATE TABLE publish_logs (
    id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(id),
    rewrite_history_id INT REFERENCES rewrite_history(id),
    platform VARCHAR(50),  -- wechat/zhihu/toutiao
    status VARCHAR(20),  -- success/failed/pending
    error_message TEXT,
    published_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT NOW()
);
```

## 8. 风险与应对

### 8.1 技术风险

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| **平台反爬** | URL采集失败率高 | 使用代理池、模拟浏览器 |
| **AI改写质量差** | 用户需要大量手动调整 | 持续优化Prompt、收集用户反馈 |
| **平台API变化** | 发布功能失效 | 监控API变化、快速适配 |
| **高并发性能问题** | 系统响应慢 | 使用缓存、异步任务队列 |

---

### 8.2 商业风险

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| **版权纠纷** | 法律风险 | 只做按需采集，不存储全文 |
| **平台政策变化** | 发布功能受限 | 多平台覆盖，降低单一平台依赖 |
| **竞品竞争** | 用户流失 | 持续优化体验，建立用户粘性 |
| **盈利困难** | 商业模式不清晰 | 先积累用户，后探索商业化 |

---

### 8.3 运营风险

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| **用户增长慢** | 无法达到目标 | 内容营销、社群运营、KOL合作 |
| **用户留存低** | 产品价值不足 | 持续优化核心功能、增加粘性功能 |
| **客服压力大** | 用户满意度下降 | 完善帮助文档、自动化客服 |
